# 3-Point Teardown Templates

## Andela — 3-Point Teardown (Draft)

### 1) Onboarding Handoff Automation
- **Problem:** Multi-team handoffs create delays and visibility gaps.
- **Fix:** Auto-triggered onboarding workflow with status checkpoints and escalation rules.
- **Outcome target:** Faster activation cycle and fewer dropped handoffs.

### 2) Placement Pipeline Visibility
- **Problem:** Pipeline tracking is fragmented across tools/channels.
- **Fix:** Unified internal dashboard for candidate stage movement, bottlenecks, and aging.
- **Outcome target:** Better forecast accuracy and intervention speed.

### 3) Exception Routing + SLA Alerts
- **Problem:** Issues escalate late, impacting delivery quality.
- **Fix:** Rule-based exception detection with SLA breach alerts to relevant owners.
- **Outcome target:** Lower incident resolution time and improved client confidence.

## Sabi — 3-Point Teardown (Draft)

### 1) Merchant/Order Exception Routing
- **Problem:** Exceptions are triaged manually and inconsistently.
- **Fix:** Automated classification and assignment by severity/type.
- **Outcome target:** Reduced processing delays and fewer unresolved exceptions.

### 2) Support Triage Automation
- **Problem:** High-volume support requests dilute team efficiency.
- **Fix:** Intent-based routing with priority tagging and owner assignment.
- **Outcome target:** Faster first response and improved resolution consistency.

### 3) Ops Visibility Dashboard
- **Problem:** Teams lack real-time performance visibility.
- **Fix:** KPI layer for exception volume, response times, and throughput by queue.
- **Outcome target:** Better daily execution decisions and accountability.
